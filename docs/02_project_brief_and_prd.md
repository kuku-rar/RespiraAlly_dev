## 第 3 部分：商業目標 (Business Objectives) - 「為何做？」

*在投入資源前，我們必須清晰地定義問題、目標使用者和商業價值，避免開發出無人需要的產品。*

| 區塊 | 內容 |
| :--- | :--- |
| **1. 背景與痛點** | **使用者**: 主要為65歲以上、患有COPD的高齡長者，他們常獨居、記憶力衰退且對新科技有焦慮感。次要使用者為醫院的呼吸治療師，他們面臨人力短缺、工作負荷大、病患數據零散的困境。<br><br>**痛點**: 患者難以規律服藥與復健，且有孤獨感；治療師難以進行持續性追蹤與衛教，且數據整理耗時。COPD作為台灣十大死因之一，現行照護模式已遇瓶頸，亟需科技輔助來延伸照護場景，提升醫療品質與效率。 |
| **2. 策略契合度** | 此專案旨在打造一個可擴展的慢性病管理解決方案，驗證AI語音機器人在高齡長照領域的應用價值與市場潛力，完全契合公司在「智慧醫療」與「AI應用落地」的戰略方向。 |
| **3. 成功指標 (Success Metrics)** | *我們如何衡量成功？定義 1-3 個可量化的關鍵績效指標 (KPI)。* <br> - **主要指標 (領先)**: 每日完成健康回報的患者比例 > 70%。<br> - **次要指標 (領先)**: 患者每週與AI助理的平均互動次數 > 3次。<br> - **主要指標 (落後)**: 半年內，使用者CAT分數平均下降幅度。<br> - **次要指標 (落後)**: 呼吸治療師平均花在每位病患資料整理上的時間減少 > 30%。 |

## 第 5 部分：範圍與限制 (Scope & Constraints)

*明確定義邊界是專案成功的關鍵。*

| 區塊 | 內容 |
| :--- | :--- |
| **功能性需求 (In Scope)** | - **LINE LIFF 健康日誌**: 每日健康追蹤表單。<br>- **臨床問卷系統**: MMRC & CAT 問卷。<br>- **呼吸治療師儀表板**: 病患列表、個人化數據儀表板、趨勢圖、備註欄。<br>- **整合式聊天室**: 內嵌於儀表板，支援人工接管。<br>- **中文 AI 助理**: 支援文字與語音互動 (STT -> LLM(RAG) -> TTS)。<br>- **後台管理頁面**: CRUD 操作資料庫、即時更新 RAG 語料。 |
| **非功能性需求 (NFRs)** | - **性能**: API 回應時間 < 500ms，頁面載入 < 2秒。<br>- **安全性**: 密碼需使用 bcrypt 加密，API 需通過 JWT 驗證。<br>- **可用性**: LIFF 介面字體與按鈕需針對高齡使用者優化。<br>- **可靠性**: AI 服務與後端服務需解耦，一方故障不影響另一方。 |
| **不做什麼 (Out of Scope)** | - **數字人助理**: 不開發擬人化影像的 AI 助理。<br>- **家屬帳號系統**: 不提供給家屬的獨立登入介面。<br>- **自動開立醫囑**: AI 不提供任何診斷或處方建議。<br>- **串接醫院 HIS/EMR**: 不與醫院內部電子病歷系統進行資料交換。 |
| **假設與依賴** | - **假設**: 目標使用者熟悉 LINE 基本操作，並願意每日回報健康狀況。<br>- **依賴**: 專案高度依賴 LINE LIFF 平台、外部 LLM/**繁體中文 STT/TTS** 服務的穩定性，以及醫療專家提供的衛教知識庫。 |
